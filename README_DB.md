# 🗂️ 구미/경북권 관광 데이터베이스(SQLite) 구조 안내

원본 TourAPI 4.0 JSON 파일 8개를 통합 데이터베이스(`ggb_tour_data.db`)로 구축하면서 진행한 **구조적 변경 사항(정규화)** 및 **테이블 설계 개요**입니다[cite: 12]. 팀원 분들은 개발 시 아래 구조를 참고하여 쿼리를 작성해 주세요.

---

## 🛠️ 1. 구조적 변경 및 데이터 정제 사항 (JSON ──> DB)

데이터의 알맹이(내용)는 원본과 100% 동일하지만, 관계형 데이터베이스(RDB)의 효율성을 극대화하기 위해 다음과 같이 구조를 변경했습니다[cite: 11].

1. **상하 관계 분리 (데이터 정규화)**
   * **기존 JSON:** 파일마다 상위 정보(`region`, `contentType`, `contentTypeId`)가 들어있고, 그 아래에 수백 개의 장소(`items`)가 속해 있는 독립된 구조였습니다[cite: 11]. 이대로 저장하면 '구미_경북권', '관광지'라는 긴 문자열이 모든 장소마다 중복 저장되어 용량을 낭비하게 됩니다.
   * **변경 후 DB:** 상위 카테고리만 따로 모은 `content_types` 테이블과, 실제 1,667개의 장소만 모은 `tour_items` 테이블로 **분리**했습니다[cite: 11, 12]. 그리고 장소 옆에는 긴 글자 대신 카테고리 ID 번호(`content_type_id`)만 적어서 두 표를 연결했습니다[cite: 11].
2. **좌표 데이터 타입 변환 (String ──> REAL)**
   * 명세서 주의 사항에 따라, 기존 JSON에서 문자열(`"128.438..."`)로 저장되어 있어 수치 계산이 불가능했던 위도/경도(`mapx`, `mapy`) 데이터를 연산 및 정렬이 가능한 **실수형 숫자(`REAL`)** 타입으로 정제하여 적재했습니다[cite: 11, 13].

## 📊 2. 간략한 테이블 구조 정의서

### ① `content_types` 테이블 (부모 표)
* **개요:** 데이터의 상위 카테고리(대분류 기준)를 정의하는 마스터 테이블입니다[cite: 11]. (총 8줄)[cite: 12]
* **구조:**
  * `content_type_id` (정수형, PK): 카테고리 고유 번호 (12: 관광지, 39: 음식점 등)[cite: 11]
  * `region` (문자열): 수집 권역명 (`구미_경북권`)[cite: 13]
  * `content_type` (문자열): 카테고리 한글명[cite: 11]

### ② `tour_items` 테이블 (자식 표)
* **개요:** 구미/경북권 권역의 실제 POI 장소 정보가 담긴 메인 테이블입니다[cite: 12]. (총 1,667줄)[cite: 12]
* **구조:**
  * `content_id` (문자열, PK): 각 장소의 고유 식별 번호[cite: 11]
  * `content_type_id` (정수형, FK): 이 장소의 카테고리 번호 (`content_types` 테이블과 연결되는 고리)[cite: 11]
  * `title` (문자열): 장소명[cite: 11]
  * `addr1` / `addr2` (문자열): 도로명/지번 주소 및 상세 주소[cite: 11]
  * `zipcode` / `tel` (문자열): 우편번호 및 전화번호[cite: 11]
  * `mapx` / `mapy` (실수형, **REAL**): **정제된 경도 / 위도 좌표 숫자**[cite: 11]
  * `first_image` / `first_image2` (문자열): 대표 이미지 및 썸네일 URL 링크[cite: 11]
  * *기타 TourAPI 원본 필드 고유 적재 (`cat1~3`, `created_time`, `modified_time` 등)*[cite: 11]

## 🔗 3. 테이블 간의 관계 (Relationship)

두 테이블은 `content_type_id`라는 공통 분모(ID 번호)로 묶여 있습니다[cite: 11]. 
프로젝트 소스코드에서 데이터를 조회할 때는 이 번호를 기준으로 두 테이블을 합쳐서(`JOIN`) 사용하시면 됩니다[cite: 11].

* **연결 구조 예시:**
  ```text
  [tour_items] 장소 표               [content_types] 카테고리 표
  - 타이틀: 검성지 생태공원[cite: 13]       
  - content_type_id: 12  ───(연결)───> - content_type_id: 12[cite: 11]
                                       - content_type: 관광지[cite: 11]
